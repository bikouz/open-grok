# Open Grok v0.1.220-open-grok.1

This is the first community release of **Open Grok: Grok Build with ChatGPT
Codex optimizations**.

## Highlights

- Adds the public `open-grok` command so the fork can coexist with an upstream
  `grok` installation.
- Ports the Codex persistent JavaScript Code Mode runtime, including native
  freeform `exec`, `wait`, nested `tools.*` calls, yielded/background cells,
  and Code Mode Only tool exposure.
- Adds GPT-5.6 Sol with a 353,000-token context window, Codex Responses routing,
  backend-hosted search, and mandatory `code_mode_only` behavior.
- Adds ChatGPT Codex OAuth with isolated credentials in
  `~/.opengrok/codex-auth.json`, including browser, device-code, refresh, logout,
  and fail-closed account-boundary behavior.
- Shows xAI billing and Codex quota windows together in `/usage`, while keeping
  failures independent.
- Makes hosted search provider-aware: xAI models retain xAI web/X search and
  Codex models receive OpenAI `web_search` without cross-provider credential or
  tool leakage.
- Disables the upstream automatic updater. Open Grok releases are installed
  explicitly from [`mweinbach/open-grok`](https://github.com/mweinbach/open-grok).

See the repository's
[`docs/code-mode-port.md`](https://github.com/mweinbach/open-grok/blob/v0.1.220-open-grok.1/docs/code-mode-port.md)
and
[`docs/codex-provider-port.md`](https://github.com/mweinbach/open-grok/blob/v0.1.220-open-grok.1/docs/codex-provider-port.md)
for the pinned Codex baseline and compatibility details.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.1/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest, then atomically installs
only `open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin` unless
`OPEN_GROK_BIN_DIR` is set. It never replaces or creates `grok` or `agent` links.

## Release assets

| Asset | Purpose |
| --- | --- |
| `open-grok-macos-aarch64` | Raw Apple Silicon macOS executable |
| `open-grok-macos-aarch64.sha256` | SHA-256 digest used by the installer |
| `install.sh` | Version-aware installer with checksum verification |
| `LICENSE` | Apache License 2.0 text distributed with the binary |
| `THIRD-PARTY-NOTICES` | Ported and third-party attribution distributed with the binary |

The executable is stripped and ad-hoc signed. It is not Apple Developer ID
signed or notarized, so macOS may display an unidentified-developer or
Gatekeeper warning. Users who require notarized software should build from
source and sign it under their own security policy.

## Compatibility

Open Grok defaults user-level state to `~/.opengrok`, honors the
`OPENGROK_HOME` override, and uses project-local `.opengrok` configuration.
It does not fall back to upstream Grok Build state. Codex OAuth is isolated in
`~/.opengrok/codex-auth.json` by default.
