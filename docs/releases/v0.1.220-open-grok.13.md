# Open Grok v0.1.220-open-grok.13

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Agent Swarms

- The new `agent_swarm` tool launches coordinated foreground subagents for
  homogeneous independent work. It validates the full batch before launch,
  supports ordered item templates and explicit subagent resumes, caps a swarm
  at 128 members, and returns results in stable input order.
- Swarms launch up to five members immediately and then ramp one member every
  700 ms, with an optional concurrency cap. Provider rate limits suspend the
  live child and requeue a same-session retry ahead of resumes and new work.
- Rate-limited members use 3 s, 6 s, 12 s, and progressively longer per-member
  eligibility delays. The scheduler also shrinks concurrency during repeated
  429s, recovers after a quiet window, and fails the final unfinished member
  instead of leaving a swarm suspended indefinitely.
- Subagent resumes retain their original role and exact provider/model route
  when available. Brief child responses receive one best-effort continuation
  request so parent-agent handoffs include useful technical context.

## Swarm controls and interface

- `/swarm`, `/swarm on`, and `/swarm off` control persistent manual swarm mode.
  `/swarm <task>` enables swarms for one task and exits automatically afterward.
- Settings include a persistent **Swarm mode** toggle backed by
  `[ui].swarm_mode`, and active sessions show a `swarm` badge.
- The TUI groups each swarm into one expandable scrollback card with ordered
  member rows, aggregate queued/running/completed/failed/cancelled counts, live
  duration and context details, and explicit waiting/retrying rate-limit state.
- Swarm members keep the existing flat subagent tree and cannot launch nested
  `task` or `agent_swarm` calls.

## Response presentation

- Responses-style display citations stay hidden instead of surfacing provider
  metadata as user-visible citation blocks.
- Codex thinking summaries are restored in the TUI while preserving the
  existing provider-aware response handling.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.13/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to the selected Open Grok bin directory under
`OPENGROK_HOME`. It does not replace or create `grok` or `agent` commands,
and it does not read or write `~/.grok`.
