# Open Grok v0.1.220-open-grok.15

Open Grok is Grok Build with ChatGPT Codex and Kimi provider support, packaged
as the independent `open-grok` command with isolated state under
`OPENGROK_HOME`.

This release brings the fork forward with the compatible feature set from
upstream Grok Build commit `7cfcb20`, while preserving Open Grok's provider,
Code Mode, credential, and filesystem isolation.

## Editing and terminal parity

- Text inputs across dialogs, search bars, settings, pickers, personas,
  extensions, dashboard views, and text areas now use the shared canonical
  editor, including improved paste, readline, and terminal behavior.
- Diff and file-viewer highlighting handles multi-line constructs correctly.
- Prompt presentation latency is bounded, prompt editing no longer causes
  enter-key scroll jank, and an optional setting snaps a sent prompt to the top.
- Dashboard exit restores the previous agent, and the minimal `/btw` side panel
  now participates in the live region.

## Session and agent lifecycle

- Session churn and agent-registry counts are exposed and gated by new
  lifecycle tests, including a dhat heap steady-state soak.
- Orphaned tool-call updates are cleared at turn end, canceled tasks no longer
  emit wakeups, and dashboard peek lease transitions are hardened.
- Waiting agents keep a persistent status row with quieter turn markers, and
  leader rosters remain visible when the local agent map is empty.
- `/summarize` is available as an alias for `/recap`, compaction uses automatic
  tool choice, and queued prompt draining uses the simplified lifecycle API.

## Security and trust

- `web_fetch` blocks non-public addresses by default. Local access requires an
  explicit opt-in and an explicit loopback host; DNS rebinding and private
  address ranges remain blocked.
- Remote workspace servers no longer implicitly trust repository LSP config,
  preventing unapproved repository code execution.
- MCP rejects ambiguous qualified tool IDs, plugin Git operands are hardened,
  and sensitive edit targets receive additional policy checks.
- Project rules are discovered under `<root>/rules`, while project roles and
  personas remain gated by workspace trust.
- `CLAUDE.md`/`AGENTS.md` content can no longer inject System-Reminder XML tags.

## Models, authentication, and privacy

- The bundled xAI default is now `grok-4.5`. Open Grok additionally fixes
  multi-provider fallback ordering so Kimi or Codex catalog entries cannot
  accidentally replace the bundled default when no preference is configured.
- MCP OAuth forwards RFC 9207 `iss` through token exchange, and authentication
  refresh failures trigger the intended retry path.
- Direct API tool calls send the Open Grok client identifier, chat attachment
  IDs reach gateway prompts, and preview/computer lifecycle provenance is
  forwarded consistently.
- Coding-data sharing defaults to opt-out until the server preference applies;
  `/privacy` and the user guide clarify the independent privacy controls.

## Tools and integrations

- Local Bash snapshots the login-shell environment, aliases, and functions
  once, improving parity without sourcing startup files for every command.
- Server-native tool descriptions template hardcoded parameter names, and Bash
  tool descriptions now reach tool-usage cards.
- Marketplace default skills are no longer auto-installed, and obsolete
  marketplace-managed installs are removed.
- Rate-limit and Cloudflare retry errors now present concise actionable text
  instead of internal prefixes or HTML response bodies.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.15/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to the selected Open Grok bin directory under
`OPENGROK_HOME`. It does not replace or create `grok` or `agent` commands,
and it does not read or write `~/.grok`.
