# Open Grok v0.1.220-open-grok.17

Hotfix for v0.1.220-open-grok.16.

## Session creation fixed

v0.1.220-open-grok.16 shipped the new client `x_search` tool wired into the
agent builder without registering the implementation in the tool registry.
Agent construction then failed Requirements validation ("GrokBuild:x_search
not found in registry") for every session where xAI was signed in — X search
defaults on — so new sessions could not be created: messages would not send
and model or reasoning-effort changes had nothing to apply to.

The tool is now registered, and a builder regression test finalizes a real
toolset with x_search enabled — the exact path that broke — so a pushed but
unregistered tool can no longer ship.

All v0.1.220-open-grok.16 features are otherwise unchanged; see its notes
for the per-provider web search source, X search, and Code Mode plan-approval
fixes.

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```
