# Open Grok v0.1.220-open-grok.18

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release adds Fireworks AI as a fourth built-in provider with a curated
model lineup, and cleans up how saved API keys read in Settings.

## Fireworks AI provider

- Four curated models over Fireworks' OpenAI-compatible Chat Completions API:
  **GLM 5.2** (1M context), **GLM 5.2 Fast** (the low-latency router, same 1M
  window), **DeepSeek V4 Pro** (1M), and **Kimi K2.7 Code** (256K).
- `/login fireworks` (also in the `/login` picker) opens a focused, masked
  key editor; the key persists in the owner-only `auth.json` under an
  isolated `fireworks::api_key` scope, fully separate from xAI, Codex, and
  Kimi credentials. `FIREWORKS_API_KEY` works as an explicit environment
  override.
- Saving a key queries the Fireworks models API to refresh curated model
  metadata (context windows). The wire response can enrich the curated
  entries but can never add or remove models, and the UI-stored key is only
  ever sent to the official `api.fireworks.ai` host.
- Live key changes follow the same fail-closed contract as Kimi: saving or
  clearing the key cancels in-flight Fireworks subagents, refreshes the
  catalog partition, and rebinds loaded Fireworks sessions so the new
  credential applies without a restart — queued prompts stay held until the
  session is rebound.
- Fireworks sessions get the client-executed xAI web search tool by default
  when xAI is signed in, with a dedicated "Web search — Fireworks AI"
  Settings row (xAI, Perplexity, or none).

## Settings polish

- Saved API keys now display as "saved" instead of "stored in UI" across
  Settings and the `/login` provider picker.

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```

Or from a running session: `/update`.
