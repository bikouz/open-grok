# Open Grok v0.1.220-open-grok.22

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release makes workflows interruptible and steerable while they run —
the two gaps the first production runs exposed.

## Workflows run in the background by default

Launching a workflow now returns immediately with a run id; the conversation
stays free while agents work. A follow-up message never ends a running
workflow — the model answers normally and can watch or steer the run:

- Progress is pollable through the task output tool, and the run appears in
  the tasks pane with a live progress log
  (`<session>/workflows/<run_id>/progress.log`).
- The kill tool and the tasks-pane kill button interrupt a run gracefully:
  the script stops, in-flight agents cancel, completed steps stay journaled,
  and the recorded result carries the resume hint.
- Completion wakes the session with the full result automatically.
- `run_in_background: false` restores the blocking inline-result behavior.

## Go back to a specific point

Resume is no longer all-or-nothing exact matching:

- Journal keys ignore display-only `label`/`phase` changes (existing
  journals migrate transparently).
- `resume_mode: "positional"` replays completed steps by call position, so
  an edited script still reuses its finished work when the call structure is
  unchanged.
- `resume_through: <phase title | agent label | call index>` replays results
  through that point and re-runs everything after it — interrupt, edit,
  resume from exactly where you want.

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```

Or from a running session: `/update`.
