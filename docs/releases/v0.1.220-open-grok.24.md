# Open Grok v0.1.220-open-grok.24

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release fixes a silent data-loss bug in `apply_patch` when one patch
touches the same file more than once.

## Repeated-path hunks chain instead of clobbering

A patch containing two `*** Update File:` sections for the same file derived
both from the on-disk original, so the second write silently erased the
first hunk's changes. Hunks now read through an in-memory overlay of the
content earlier hunks would leave behind — matching upstream codex's
sequential-apply semantics while keeping application atomic:

- A second Update to the same file builds on the first — both changes land.
- An Update after an Add in the same patch derives from the added content
  instead of failing to read a file that isn't on disk yet.
- Edits chain through a `*** Move to:` rename — the destination receives
  every change, the source is removed.
- An Update targeting a file a previous hunk deleted fails with a precise
  error (`file was deleted by an earlier hunk in this patch`) and, as
  always, nothing is written when any hunk is invalid.

## xAI: 426 "version (none)" on cross-provider subagents

xAI's API now requires a parseable client version on every request. Open
Grok's session client version is legitimately absent on cross-provider
paths — most visibly, a subagent spawned from a Codex-model session with a
Grok model override inherited no version at all, so every such child died
with `426 Upgrade Required: Your Grok CLI version (none) is outdated` while
Codex and Fireworks agents ran normally.

Every xAI request now carries a client version: the configured session value
when present, otherwise this build's own version, normalized to its base
semver (the `-open-grok.N` suffix is stripped for the gate's parser).

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```

Or from a running session: `/update`.
