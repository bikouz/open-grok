# Open Grok v0.1.220-open-grok.25

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release syncs the upstream `xai-org/grok-build` snapshots through
`a881e670` (upstream v0.2.106), bringing in the new Stop-hook gate, model
auth providers, in-place scheduled-task updates, and clipboard reliability
fixes — with all Open Grok features (workflow orchestration, Antigravity
subagents, provider-boundary auth hardening) re-verified on top.

## Stop hooks can hold the agent to a finish line

A `Stop` hook now runs when the agent tries to end its turn and can block
completion until a condition holds — for example, keep working until the
test suite passes — feeding its reason back to the model. Session-end paths
(channel closed, shutdown) dispatch through the same gate, and a runnable
`stop-verify` example ships under the hooks examples.

## Model auth providers (`[auth_provider.<name>]`)

A model can opt in to a named auth provider: a credential-helper command
that prints a fresh bearer token (bare or `{access_token, expires_in}`
JSON), which Open Grok mints, caches in memory only, and rotates for that
model's requests. Tokens never touch `auth.json`; the helper owns durable
storage and OAuth2 refresh. Declared in trusted config layers only.
Open Grok's provider-aware credential scoping (endpoint-trust gating of
session credentials and `x-grok-*` headers) is preserved on top of the new
resolution chain.

## Scheduled tasks and subagent runtime

- Scheduled tasks can be updated in place; one-time tasks are retired in
  favor of background commands.
- Subagent completion output is now capped (`completion_output_cap`), and
  spawn depth is an explicit runtime override — both threaded through Open
  Grok's workflow, agent-swarm, and Antigravity subagent backends.

## Terminal and clipboard reliability

- Copies always write a backup file (`~/.opengrok/last-copy.txt`) so text
  remains recoverable when the terminal clipboard fails.
- New `GROK_CLIPBOARD_NO_OSC52` env var stops OSC 52 clipboard sequences
  from appearing as garbage in terminals that don't support them.
- Syntax highlighting in `--minimal` mode is now visible on light
  terminals.

## Also in this sync

- Permission auto-mode classifier rework with typed classifier outcomes.
- Session storage and persistence rework (pending-append outcomes, durable
  JSONL changes) merged with Open Grok's remote-sync semantics intact.
- Upstream's expanded hooks engine (canonical event dedup, command/HTTP
  handler types, output truncation) and refreshed hooks documentation.
