# Open Grok v0.1.220-open-grok.26

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release syncs the upstream `xai-org/grok-build` snapshot `3af4d5d3`
(upstream v0.2.107) and **replaces Open Grok's custom JavaScript workflow
engine with upstream's new native workflow runtime** — Rhai-scripted,
journaled, budget-accounted, with a dedicated TUI surface. All other Open
Grok features (Antigravity subagents, multi-provider auth, provider-boundary
privacy) are re-verified on top.

## Native workflows replace the JS workflow tool (breaking)

The `workflow` tool is now upstream's native runtime: a deterministic Rhai
interpreter (`xai-workflow` crate) driving real subagents through the
session's workflow manager, with an append-only journal, agent-call budgets
(default 128, max 1024), JSON-Schema output contracts, and first-class
pause/resume/cancel.

Upstream gates workflows off by default and opts sessions in through its
remote feature flag; Open Grok keeps them **on by default** (workflows have
been generally available here since `.20`). Disable with
`[workflows] enabled = false` or `GROK_WORKFLOWS=0`. Note that this setting
also selects the `/goal` driver: with workflows on (the Open Grok default),
`/goal` rounds are driven by the host-owned workflow engine rather than the
legacy `update_goal` tool.

What's new with it:

- **`/workflows` overlay and inline workflow cards** — live phases,
  per-agent progress and tokens, and management actions
  (`/workflow pause|resume|stop|save <name>`).
- **A saved-workflow registry** — builtin `deep-research`, project
  `.opengrok/workflows/*.rhai` (folder-trust gated), and user
  `~/.opengrok/workflows/*.rhai`; registered workflows surface as their own
  slash commands.
- **Completion turns** — a finished run injects its result back into the
  conversation automatically; no polling.
- **Antigravity subagents work inside workflows**: `agent()` calls with
  `model: "antigravity:<model>"` route through the same subagent
  coordinator as `task`/`agent_swarm`.

Breaking changes from the old Open Grok workflow tool:

- Scripts are **Rhai, not JavaScript**. The `export const meta` /
  `agent()/parallel()/pipeline()` JS surface is gone; scripts start with
  `let meta = #{ … }` and use the Rhai host functions
  (`agent`, `parallel`, `phase`, `log`, `complete`, `pause`, `await_user`,
  `budget`, scratch/template/diff helpers).
- Budgets count **agent calls**, not tokens (`agent_budget` replaces
  `token_budget`).
- `resume_mode: "positional"` and `resume_through` are gone; resume replays
  the byte-identical script same-process only, and process restarts mark
  runs interrupted.
- `OPENGROK_WORKFLOW_MAX_CONCURRENCY` is retired (concurrency is managed by
  the runtime); `OPENGROK_SUBAGENT_TIMEOUT_MS` still governs task/swarm and
  Antigravity subagents.
- Runs are always background; the virtual-task shim that gave workflow runs
  bash-task parity is removed along with the JS engine.

Open Grok also ships its own **`create-workflow`** and
**`import-claude-workflow`** skills (upstream delivers these outside the
source tree via a bundled skill cache). They are extracted to
`~/.opengrok/skills/` on first run — created only if absent, never
overwriting your edited copies. `import-claude-workflow` walks through
porting Claude Code JS workflow scripts to the Rhai runtime, including the
constructs with no equivalent (`pipeline()` staging, token budgets,
positional resume).

## Also in this sync

- **Worktree auto-GC** — throttled GC on startup (age-based on Linux,
  dead-only elsewhere), kind-aware TTL knobs, macOS process-CWD scanning and
  Unix PID liveness for GC guards, optional rebuild, and stale git
  registration cleanup.
- **`/usage` in the TUI** — per-session token and dollar usage, integrated
  with Open Grok's combined xAI + Codex usage view.
- **`/doctor`** — read-only doctor surfaced in the TUI, plus an SSH-wrap
  setup fix.
- **Pager** — `[ui].combine_queued_prompts` batches queued follow-ups,
  backgrounding standardized on Ctrl+B, screen-mode-aware actions, external
  editor for minimal prompts, image-paste recovery over `grok wrap` on
  headless remotes, and no more stacked "Worked for" markers on parked
  turns.
- **Models** — `max` as a distinct reasoning-effort tier (Open Grok's
  `ultra` tier preserved alongside), model providers plumbing, and side
  model calls with their own conversation ids.
- **Goal mode** — upstream's goal classifier/planner rework; with workflows
  enabled, `/goal` is driven by the workflow engine.
- **Security hardening** — permission prompts on environment-dumping `ps`
  variants, `kubectl` no longer runs arbitrary kubeconfig credential
  plugins unprompted, `env -S` operand peeling in the Bash permission gate,
  `rg --pre` blocked in auto-mode, and unauthorized-RCE fixes for abused
  safe commands.
- **Sessions** — working-directory relocation primitives (sessions resume
  when the cwd moves), rewind accepts a target response id, and recap rides
  the parent turn's prompt cache.
- **Scheduler** — durable deletion and a lifecycle version clock.
- **Skills** — the shell stops overwriting user skills, and markdown under
  `skills/` directories is read untruncated. Upstream moved its bundled
  skills out of the source tree; Open Grok keeps shipping them
  (`help`, `create-skill`, `code-review`, `imagine`, `best-of-n`,
  `check-work`, plus new `agent-swarm` and workflow skills) via startup
  extraction to `~/.opengrok/skills/` — created only when absent, never
  overwriting your copies.
- **Voice** — silent-mic failure diagnosis (macOS permission) with a new
  doctor/terminal-setup section.
- **Sandbox** — Landlock applies without a controlling TTY.

All Open Grok features were re-merged and preserved on top: Antigravity
(agy) subagents (including inside workflow `agent()` calls), agent swarm,
X search and view-image tools, multi-provider auth and usage (Codex, Kimi,
Fireworks), feedback privacy stripping at the provider boundary, and
`.opengrok` branding throughout (workflow registries discover from
`.opengrok/workflows/`).
