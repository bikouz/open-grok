# Open Grok v0.1.220-open-grok.3

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command.

## Fixes

- Sends the leading harness prompt to ChatGPT Codex through the top-level
  Responses `instructions` field, matching codex-rs. Codex requests no longer
  contain the unsupported `system` message role.
- Preserves later contextual instruction messages in place as `developer`
  messages and keeps explicit `/responses/compact` instructions authoritative.
- Keeps xAI Responses requests unchanged by the Codex-specific projection.
- Makes the selected model catalog authoritative for the TUI context-window
  denominator. Switching from Grok Build to a Codex model now replaces a stale
  `500K` total immediately with the live Codex window (or the embedded `353K`
  fallback) without waiting for another successful model turn.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.3/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs only
`open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`, unless
`OPEN_GROK_BIN_DIR` is set. It does not replace or create `grok` or `agent`
commands.

The executable is stripped and ad-hoc signed. It is not Apple Developer ID
signed or notarized, so macOS may display an unidentified-developer or
Gatekeeper warning.

## Compatibility and provenance

Open Grok does not fall back to upstream Grok Build state. Credentials,
settings, sessions, caches, skills, plugins, and project configuration remain
isolated under `.opengrok` / `OPENGROK_HOME`.

Open Grok is a community fork of Grok Build and is not affiliated with,
sponsored by, or endorsed by xAI, SpaceXAI, or OpenAI. Codex behavior is ported
from pinned revisions of [`openai/codex`](https://github.com/openai/codex); see
[`docs/code-mode-port.md`](https://github.com/mweinbach/open-grok/blob/v0.1.220-open-grok.3/docs/code-mode-port.md),
