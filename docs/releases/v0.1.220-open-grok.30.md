# Open Grok v0.1.220-open-grok.30

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

Upstream sync release: xai-org/grok-build snapshot 69f0ba88 (v0.2.111),
ported with every fork divergence preserved.

## Shared subagent coordinator actor

Subagent lifecycle — pending/active/completed state, waiters, the
foreground await budget, buffered completions, and cancellation — moved
out of the shell into one shared coordinator actor. The shell now plugs a
child runner into that actor, and every kind of child (task, swarm member,
workflow agent, Antigravity CLI run) gets identical result delivery,
auto-wake, and `resume_from` behavior. Antigravity subagents ride the new
actor with the same semantics as before: the entry stays pending for the
run (there is no child session), kill/cancel keep working through the run
token, and resume still continues the CLI conversation via the id stored
in the durable subagent metadata.

## Version policy, the Open Grok way

Upstream replaced the old single minimum-version gate with a four-knob
policy: soft `minimum`/`maximum` steer the updater, hard
`required_minimum`/`required_maximum` (config layers or env, fail-open)
gate startup for managed deployments. Open Grok ports its fork floor
semantics into the new policy: a bare floor such as `0.1.220` is satisfied
by a same-core fork build (`0.1.220-open-grok.N`), the updater never skips
or rewrites a fork release target into a nonexistent bare tag, and every
message says `open-grok update`. Version bounds resolve from local
config/env only — remote settings stay dropped.

## `/tutorial`

An opt-in onboarding tour of the pager: nine short chapters covering
first prompts, attach/paste, navigation, slash commands, worktrees, plan
mode and permissions, and customization — rebranded for Open Grok.

## Resume by title

`open-grok --resume <value>` and `-r` in headless mode now accept a
session title as well as an ID. Non-UUID values match session titles for
the current directory case-insensitively; a single manually renamed
session wins over auto-generated duplicates, and ambiguous titles error
with the matching IDs. Scripts should keep passing IDs.

## Protected global hook sources

`$OPENGROK_HOME/hooks/` and the new `hooks-paths` registry are protected
under the sandbox write-deny profiles, with symlink refusal and pinned
parent directories, so a sandboxed agent cannot retarget the hooks the
next session will load. Project hooks remain gated by folder trust.

## Everything else from v0.2.111

- Custom provider gateways: per-model query params and
  `env_http_headers` (header values resolved from env at request time, so
  secrets stay out of persisted state).
- OIDC token refresh before spawning shell children.
- `workspace.tasks_snapshot` RPC — clients rebuild background/scheduled
  task trays on attach or reconnect.
- Doctor: tmux clipboard/passthrough fixes with TUI-applied remediation.
- A "Voice shortcut" setting to disable the Ctrl+Space/F8 chord without
  hiding the other voice surfaces.
- Marketplace git-URL validation, app-builder archive size caps, ACP
  terminal output recorder, workspace-server `/ready` failure dwell, and
  queued-prompt row edit controls.

Fork invariants verified through the port: workflows stay enabled by
default with remote feature flags ignored wholesale (upstream's new
privacy-notice and upsell surfaces resolve from those flags and are
therefore inert here), provider credentials and export boundaries stay
provider-local across the new spawn path, Codex web search keeps
`external_web_access: true` on the wire, and the nine bundled skills
still ship in-binary.
