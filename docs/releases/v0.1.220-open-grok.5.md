# Open Grok v0.1.220-open-grok.5

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Stable session resume

- Restored transcripts now paint in one stable pass instead of visibly
  rebuilding a row at a time.
- Replayed hook executions, background-task reconciliation, and scheduled-loop
  updates still restore their state, but no longer trigger intermediate
  redraws.
- Historical tool completions no longer receive a fresh live-completion accent
  flash.
- Model-catalog and MCP initialization updates continue in the background and
  become visible after the session load settles.
- The ambient small-screen hint is suppressed during a restored session so it
  cannot briefly insert and remove a status row.

## Cross-provider subagents

- Parent models may select any authenticated model in the live catalog for a
  subagent when its speed, cost, depth, or provider capabilities better fit the
  delegated task. Omitting the override still inherits the parent model.
- A GPT-5.6 Sol parent can therefore delegate to GPT-5.6 Luna, GPT-5.6 Terra,
  or an available Grok model; Grok parents can likewise delegate to available
  Codex models.
- Coordinator validation now treats xAI OAuth and Codex OAuth independently,
  matching the model picker and task schema. Child requests continue to use
  only the selected provider's isolated credentials.

This release includes all Codex OAuth, model-catalog, usage, code-mode,
reasoning-summary, Responses streaming, cache continuity, remote compaction,
web-search, updater, and `.opengrok` isolation work from
`v0.1.220-open-grok.4`.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.5/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`. It does not replace
or create `grok` or `agent` commands, and it does not read or write `~/.grok`.
