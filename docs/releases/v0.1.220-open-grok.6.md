# Open Grok v0.1.220-open-grok.6

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Provider-aware recap and memory

- Recap, memory flush, Dream consolidation, and memory-note rewriting now use
  independently resolved helper models instead of inheriting the active chat
  client by accident.
- Settings includes separate **Recap model** and **Memory model** selectors
  backed by the authenticated live model catalog. **Automatic** is the default
  and really removes any persisted override.
- Automatic Codex sessions use `gpt-5.6-terra` at medium reasoning. Automatic
  xAI sessions reuse the active Grok model at low reasoning when that effort is
  advertised by the live catalog.
- Explicit selections can cross providers in either direction, so a GPT chat
  may use Grok for recap or memory and a Grok chat may use a Codex model. Each
  helper uses its selected provider's endpoint and isolated credentials.
- Recap and memory prompts budget against the helper model's own context window.
  Reasoning models receive room for hidden reasoning while final memory-note
  output remains bounded.

## Memory across providers

- Local memory storage, FTS indexing, initial injection, and memory tools remain
  available in both xAI and Codex sessions.
- When xAI credentials and an embedding model are configured, Codex chats may
  use the independent xAI embedding route for semantic memory search. Missing
  or failed embeddings degrade cleanly to FTS.
- Embedding catalog aliases now resolve to their wire model, endpoint, and
  credential together.
- Custom xAI or Codex helper endpoints must provide their own API key or
  environment-backed credential. ChatGPT OAuth and xAI session tokens are never
  inherited by arbitrary configured hosts.

## Codex presentation defaults

- Supported Codex models default reasoning summaries to **detailed** while
  preserving explicit `auto`, `concise`, `detailed`, and `none` catalog values.
- Code Mode `read_file`, `list_dir`, and `grep_files` calls render as normal,
  descriptive TUI tool calls. Dynamic tools show their actual wire name rather
  than exposing generic raw protocol JSON.

This release includes all Codex OAuth, live model-catalog, usage, compaction,
Responses streaming, response-cache continuity, web-search, resume stability,
cross-provider subagent, updater, and `.opengrok` isolation work from
`v0.1.220-open-grok.5`.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.6/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`. It does not replace
or create `grok` or `agent` commands, and it does not read or write `~/.grok`.
